API

Application Programming Interface ehk API ehk rakendusliides võib tunduda maagilise tähekombinatsioonina, millest oled varem palju kuulnud, ent selle tegelik sisu või vajalikkus ei pruugi sulle veel päris teada olla. Kahtlemata puutud järgmiste aastate jooksul APIdega ühel või teisel viisil, mistõttu proovime anda põgusa sissejuhatuse millega tegu on.

Mis on API?

Ennekõike on API enimlevniumaid andmevahetusvorme arvutite vahel. Internetipõhises maailmas on arendatud kõikvõimalikud desktopi- ja veebirakendused rakendused inimestele kasutamiseks. Kasutajatel ei ole enamasti erilisi probleeme teksti lugemisega siniselt või valgelt taustalt, samuti ei tohiks font ega teksti suurus osutuda eriliseks takistuseks tekstist arusaamisel. Arvutiprogrammid seejuures jäävad aga üksteisega suheldes hätta, kui nad ei tea, kust või millist infot lugeda. Vahest on sellest lihtsam mõelda kui mõne võõrkeele õppimisest - hiinakeelset intervjuud on põnev kuulata, ent kui te grammatika, häälduse ning foneetika reegleid koos sõnavaraga ei tunne, jääb kuuldav-nähtav teile arusaamatuks.

Selleks, et arvutiprogrammid üksteist mõistaksid ongi väljaarendatud standardne infovahetuse viis ning arhitektuur, mida nimetatakse REST APIks.

Põhimõte

Kõige tuntumaid viise API ehk rakendusliidese töö ilmestamiseks on nn restoraninäide. Kujutage korraks ette, et olete restoranis ning soovite tellida uhke prae. Mõelge, kuidas peate enda soovi köögile edasi andma? Kõige tõhusam viis on seda teha ettekandja abil, kes tuleb küsib teie käest, mida te süüa soovite, misjärel edastab teie palve köögile ning saabum mõnekümne minuti pärast tagasi maitsva roaga. Täpselt sama funktsiooni täidab kahe programmi vahel ka rakendusliides. Nii saab üks programm saata, pärida, muuta või kustutada infot näiteks mõnest teise programmi andmebaasist, kodulehelt või rakendusest.

Kuid kuidas?

Kuid kudias ometi kaks erinevas maailma otsas kirjutatud programmi üksteisest aru saavad?

Informatsiooni vahetamiseks on väljaarendatud erinevaid arhitektuure. Kuna tõenäoliselt puutute kokku just selle arhitektuuri stiili kasutamisega kõige varem, selgitame lähemalt ka just RESTful-tüüpi rakendusliideseid. REST (ingl k Representational State Transfer) kujutab endast tarkvaraarhitektuuri, mis defineerib, kuidas peaks API ehk rakendusliidesega infot teistele programmidele vahetama. REST arhitektuuri stiil ehk RESTful API on üks enimlevinumaid ja -kasutatumaid JavaScript Object Notation ehk JSON või XML tüüpi failide vahendamise stiile. REST API saab suhelda neljal erineval viisil: saata (post), pärida (get), muuta (put) või kustutada (delete) teise rakenduse sisu. Kuna teine progamm teab, missugusel viisil infot oodata, oskab ta lihtsalt sõne kujul olevatest võtmete väärtusest välja lugeda vajaliku info ning teha vajalikud muutused.
